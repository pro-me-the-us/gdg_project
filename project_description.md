# Multiplayer Arena Shooter - Architecture Document

## Project Overview

This project is a multiplayer 2D top-down arena shooter built in C++ using OpenGL for rendering and ENet for networking.

The primary goal of this project is to learn multiplayer game development, networking concepts, client-server architecture, and real-time game synchronization.

The game consists of 4-6 players competing inside a procedurally generated arena. Players move around using WASD, shoot projectiles, collect powerups, and attempt to achieve the highest score before the match timer expires.

# Gameplay Overview

## Objective

Players compete in a free-for-all arena.

The winner is the player with the highest score when the match timer reaches zero.

## Player Cap

* Minimum Players: 4
* Maximum Players: 6

## Match Rules

* Every player starts with full health.
* Players can move around the arena.
* Players can shoot bullets.
* Players earn points by damaging or eliminating opponents.
* When a player dies:

  * Their score is reset to zero.
  * Their health is restored.
  * They immediately respawn.
* Match ends when the timer expires.
* The player with the highest score wins.

---

# High Level Architecture

The game uses an authoritative server architecture.

The server owns all game logic and game state.

Clients are only responsible for:

* Rendering
* Input collection
* UI display

Clients never decide:

* Damage
* Health
* Score
* Deaths
* Map generation
* Collision outcomes

## Network Topology

Client ↔ Server

The same machine can act as both:

* Dedicated Server
* Local Client

# Server Architecture

## Responsibilities

The server controls:

* Match lifecycle
* Map generation
* Player movement validation
* Bullet simulation
* Collision detection
* Health calculations
* Scoring
* Powerup spawning
* State synchronization

## Server Tick Rate

45 TPS

Tick Duration:

22.22 milliseconds

---

# Server Game Loop

```cpp
while(running)
{
    ReceiveNetworkEvents();

    ProcessPlayerInputs();

    UpdatePlayers();

    UpdateBullets();

    HandleCollisions();

    HandleDeaths();

    UpdatePowerups();

    BroadcastGameState();

    SleepUntilNextTick();
}
```

Clients only transmit input commands.

```cpp
struct InputPacket
```

# Movement System

## Controls

W - Move Up

A - Move Left

S - Move Down

D - Move Right

## Characteristics

* Constant movement speed
* Speed boost powerups may temporarily modify speed

Movement is validated by the server.

---

# Shooting System

## Controls

SPACE - Fire Weapon

## Bullet Characteristics

* Travels in a straight line
* Has a fixed lifetime
* Has fixed speed
* Has fixed damage
* Destroyed on impact

---

# Collision System

## Player vs Obstacle

Players cannot move through obstacles.

## Bullet vs Obstacle

Bullet is destroyed.

## Bullet vs Player

Damage is applied.

Bullet is destroyed.

---

# Health System

Each player has health.

When health reaches zero:

* Player dies
* Score resets to zero
* Health is restored
* Player respawns immediately

All health calculations occur on the server.

# Scoring System

+1 Point

+5 Points

# Match Lifecycle

## Lobby Phase

Players connect and wait.

## Match Start

Server generates map.

Scores reset.

Timer begins.

## Gameplay Phase

Players:

* Move
* Shoot
* Collect powerups
* Earn points

## Match End

Timer reaches zero.

Leaderboard is generated.

Winner is announced.

## Reset

New map generated.

Scores reset.

Next match begins.
